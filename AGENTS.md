# Console Whisperer Deployment - gh-pages Branch

## Branch Purpose

The **gh-pages** branch is the public-facing website for **Console Whisperer**, an AI-powered mixing console setup assistant that helps live sound engineers configure channel names and colors on mixing consoles (specifically targeting **Mixing Station**) using natural language.

This branch contains:
- Marketing landing page (`index.html`)
- Download portal (`download.html`) - automatically updated by Conveyor
- Static assets (icons, images, favicons)

**Related Branch:** The `main` branch contains the packaging infrastructure (Conveyor config, GitHub Actions workflow). See that branch's AGENTS.md for packaging/release documentation.

---

## Key Technologies

- **Frontend:** HTML5, Tailwind CSS (via CDN), Google Fonts (Space Grotesk)
- **Hosting:** GitHub Pages (automatically deployed on push to this branch)
- **Download Page:** Auto-generated by [Hydraulic Conveyor](https://hydraulic.dev) during release builds

## Key Files

### Website Pages
- **`index.html`**: Main landing page with product features, pricing, and CTAs
  - Hero section with value proposition
  - Feature highlights
  - "Buy Now" button (links to Polar.sh)
  - "Download" button (links to `download.html`)
- **`download.html`**: Download portal (auto-generated by Conveyor)
  - OS detection (Windows/macOS)
  - Links to latest installers from GitHub Releases
  - Auto-updated when new versions are released

### Branding & Assets
- **`icon.svg`**: Official Console Whisperer logo (used in website and installers)
- **Favicon files**: `favicon.ico`, `favicon.svg`, `apple-touch-icon.png`, etc.
- **Screenshots**: `console-whisperer-screenshot.png` (app interface demo)

### Configuration
- **`CNAME`**: Custom domain configuration for GitHub Pages
- **`.gitignore`**: Ignores `.idea/`, `.DS_Store`, and other development artifacts

## Local Development

Since this is a static website, no build steps are required:

### Option 1: Direct File Opening
Open `index.html` directly in any modern web browser.

### Option 2: Local Server (Recommended)
Use a simple HTTP server to avoid CORS issues and test properly:

```bash
# Using Node.js
npx serve .

# Using Python
python3 -m http.server 8000

# Using Ruby
ruby -run -ehttpd . -p8000
```

Then visit: `http://localhost:8000`

### Testing Responsiveness
- Use browser DevTools to test mobile/tablet views
- Verify Tailwind breakpoints (`sm:`, `md:`, `lg:`, `xl:`)
- Check both light and dark mode (if implemented)

## Development Guidelines

### Styling Conventions
- **Tailwind-first**: Use utility classes for all styling
- **No custom CSS files**: Extend Tailwind config in `<script>` block if needed
- **Consistent spacing**: Use Tailwind's spacing scale (`p-4`, `m-8`, etc.)
- **Color palette**: Use colors that match the brand (defined in Tailwind config or inline)
- **Typography**: Space Grotesk font (loaded via Google Fonts)

### HTML Structure
- Keep semantic HTML (`<header>`, `<main>`, `<section>`, `<footer>`)
- Use descriptive class names for readability
- Maintain accessibility (proper heading hierarchy, alt text, ARIA labels)

### Asset Management
- **Icons**: Use SVG format for scalability (store in root or `/assets/`)
- **Images**: Optimize before committing (use WebP when possible)
- **Favicons**: Use `realfavicongenerator.net` or similar to generate full set

### Links and CTAs
- **Buy Now**: Points to Polar.sh product page
- **Download**: Links to `download.html` (generated by Conveyor)
- **External links**: Use `target="_blank" rel="noopener noreferrer"` for security

## Common Tasks

### Updating Landing Page Content
1. Edit `index.html` directly
2. Update hero text, features, or CTAs
3. Test locally
4. Commit and push to `gh-pages`
5. Verify on live site

### Adding New Features Section
Use Tailwind utilities to match existing design:
```html
<section class="py-16 px-4">
  <h2 class="text-3xl font-bold mb-8">New Feature</h2>
  <p class="text-lg text-gray-600">Description...</p>
</section>
```

### Updating Branding/Logo
1. Replace `icon.svg` with new logo
2. Regenerate favicons from new logo
3. Update `app-icon.svg` in `main` branch (for installer branding)

### Changing Color Scheme
Extend Tailwind config in `index.html`:
```html
<script>
  tailwind.config = {
    theme: {
      extend: {
        colors: {
          brand: {
            primary: '#yourcolor',
            secondary: '#yourcolor'
          }
        }
      }
    }
  }
</script>
```

## Important Notes

### DO NOT Edit `download.html` Manually
This file is auto-generated by Conveyor during release builds. Manual edits will be overwritten.

If you need to customize the download page:
1. Modify the Conveyor template in the `main` branch
2. Rebuild with Conveyor to regenerate `download.html`

### GitHub Pages Configuration
- **Source**: Deploy from `gh-pages` branch (configured in repo settings)
- **Custom Domain**: Edit `CNAME` file to change domain
- **HTTPS**: Automatically enforced by GitHub Pages

### Relationship with Main Branch
- `main` branch: Packaging infrastructure (Conveyor, GitHub Actions)
- `gh-pages` branch: Website content (this branch)
- Conveyor pushes to `gh-pages` during releases to update `download.html`
- Website updates (landing page) are independent of releases

## Troubleshooting

### Changes Not Showing Up
- Check GitHub Pages deployment status (repo → Settings → Pages)
- Clear browser cache (hard refresh: `Cmd+Shift+R` or `Ctrl+Shift+R`)
- Verify you pushed to `gh-pages` branch (not `main`)

### Tailwind Classes Not Working
- Ensure Tailwind CDN script is present in `<head>`
- Check for typos in class names
- Use browser DevTools to inspect computed styles

### Download Links Broken
- Verify GitHub Releases exist with installer files
- Check `download.html` has correct links (should be auto-generated)
- Test on different operating systems (Windows/macOS detection)

## Best Practices

### Before Committing
- [ ] Test in multiple browsers (Chrome, Firefox, Safari)
- [ ] Check mobile responsiveness
- [ ] Verify all links work
- [ ] Optimize images (compress, use appropriate formats)
- [ ] Validate HTML (use W3C validator if needed)

### Writing Commit Messages
Be descriptive about website changes:
```bash
# Good
git commit -m "Update hero section with new tagline and CTA button"

# Too vague
git commit -m "Update index"
```

### Performance
- Minimize external dependencies (currently just Tailwind CDN and Google Fonts)
- Lazy load images if adding many
- Use SVG for icons/logos (smaller file size, infinite scaling)
- Keep HTML clean and semantic for fast parsing
